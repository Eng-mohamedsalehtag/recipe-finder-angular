# Recipe Finder Angular

A modern, responsive web application built with Angular for discovering and exploring delicious recipes. This e-commerce style recipe finder allows users to search for recipes by category or keyword, view detailed recipe information, and manage their accounts.

## Features

- **Recipe Search**: Search for recipes using keywords or browse by categories (Pizza, Burger, Pasta, Salad, Ice Cream, Popcorn)
- **Recipe Details**: View comprehensive recipe information including ingredients, instructions, and publisher details
- **User Authentication**: Login and registration system with route guards for protected pages
- **Responsive Design**: Mobile-friendly interface using Bootstrap for consistent styling
- **Navigation**: Intuitive navbar for easy navigation between pages
- **Data Service**: Integration with Forkify API for fetching recipe data

## Tech Stack

- **Framework**: Angular 16.2.16
- **Language**: TypeScript
- **Styling**: CSS with Bootstrap classes
- **API**: Forkify API (https://forkify-api.herokuapp.com/)
- **Build Tool**: Angular CLI

- **Home**: Landing page
- **Login/Register**: User authentication

- **Product Details**: Detailed view of individual recipes

## Development server

Run `ng serve` for a dev server. Navigate to `http://localhost:4200/`. The application will automatically reload if you change any of the source files.

## Code scaffolding

Run `ng generate component component-name` to generate a new component. You can also use `ng generate directive|pipe|service|class|guard|interface|enum|module`.

## Build

Run `ng build` to build the project. The build artifacts will be stored in the `dist/` directory.

## Running unit tests

Run `ng test` to execute the unit tests via [Karma](https://karma-runner.github.io).

## Running end-to-end tests

Run `ng e2e` to execute the end-to-end tests via a platform of your choice. To use this command, you need to first add a package that implements end-to-end testing capabilities.

## Further help

To get more help on the Angular CLI use `ng help` or go check out the [Angular CLI Overview and Command Reference](https://angular.io/cli) page.
